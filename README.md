# Lugx Gaming

Современный интернет-магазин цифровых игр на Laravel.

Простой, быстрый и красивый шаблон магазина с корзиной, отзывами только от покупателей, личным кабинетом и удобной админ-панелью.

## Основные возможности

- Каталог игр с поиском, пагинацией и фильтрами
- Страница товара с описанием, скриншотами, отзывами и кнопкой «В корзину»
- Корзина (гости + авторизованные пользователи, автоматический перенос после входа)
- Отзывы: только покупатели могут писать/редактировать/удалять
- Личный кабинет: история заказов, купленные игры, редактирование профиля
- Админ-панель: CRUD игр и пользователей, отдельный вход по паролю
- Адаптивный дизайн (Bootstrap 5 + кастомные стили)

## Технологический стек

- Backend: Laravel 11/12
- Frontend: Blade + Bootstrap 5
- База данных: MySQL / SQLite (для разработки)
- Корзина: гибридная реализация (сессия + БД)
- Хранение файлов: Laravel Storage (public disk)
- Безопасность: middleware, form requests, route model binding

## Требования

- PHP ≥ 8.1
- Composer
- Node.js + npm (для сборки фронтенда, если используете Vite)
- MySQL / SQLite / PostgreSQL

## Установка и запуск (разработка)

1. **Клонируйте репозиторий**

   ```bash
   git clone https://github.com/Frimm222/lugx-gaming.git
   cd lugx-gaming
   ```
2. **Установите зависимости**
    ```bash
   cp .env.example .env
    ```
3. **Скопируйте и настройте .env**
   Отредактируйте .env:
   - APP_URL=http://localhost:8000
   - Настройки базы данных (DB_*)
   - ADMIN_PASSWORD=ваш_секретный_пароль (для входа в админку)
4. **Сгенерируйте ключ приложения**
```bash
  php artisan key:generate
```
5. **Выполните миграции и сиды**
```bash
  php artisan migrate --seed
```
(Это создаст все таблицы и добавит тестовые игры/пользователей)
6. **Создайте симлинк для хранения файлов**
```bash
  php artisan storage:link
```
7. **Запустите сервер разработки**
```bash
  php artisan serve
```
Сайт будет доступен по адресу: http://127.0.0.1:8000
8. **Вход в админ-панель**
 - Адрес: http://127.0.0.1:8000/admin/login
 - Пароль: тот, который вы указали в .env (ADMIN_PASSWORD) (можно несколько паролей)

После входа доступны:
 - /admin — дашборд
 - /admin/games — управление играми
 - /admin/users — управление пользователями
